1. What is DOM and what is the purpose of DOM?

DOM: The DOM is a W3C (World Wide Web Consortium) standard.
The W3C Document Object Model (DOM) is a platform and language-neutral interface that allows programs and scripts to dynamically access and update the content, structure and style of a document.

The W3C DOM standard is separated into 3 different parts:
(i) Core DOM - standard model for all document types.
(ii) XML DOM - standard model for XML documents.
(iii) HTML DOM - standard model for HTML documents.


Purpose of DOM: Its purpose is to provide a structured representation of an HTML or XML document so that programs or scripts can change the document structure, style, content and manipulate or interact with the page.

2. How will you select HTML elements using DOM?

There are 6 ways to select elements in a DOM using selectors:
i. getElementById()
ii. getElementsByClassName()
iii. getElementsByTagName()
iv. querySelector()
v. querySelectorAll()
vi. getElementsByName() - only that elements which has a name attribute.

3. What is event bubble?

Event bubble is a concept in the DOM.
It happens when an element receives an event and that event bubbles up (or propagates up) to its parent and ancestor elements in the DOM tree until it gets to the root element.
This is the default behavior of events on elements unless to stop the propagation.

4. What is the callback function and why will you use it?

A callback function is a function that is passed into another function as an argument which is then invoked inside the outer function to complete some kind of routine or action.


Why: We need callback function because many Javascript actions are asynchronous which means they don't really stop the program (or a function) from running until they're completed.
If I want to do asynchronous work synchronously then I need to use callback function.
Or when we want to do database related asynchronous work then we need to use callback function.
